Tu es un développeur senior React / Next.js / TypeScript.

Je veux créer une application web locale de notation pour parties de fléchettes, utilisable sur ordinateur.

Stack souhaitée :
- Next.js
- React
- TypeScript

Objectif général :
Créer une application permettant de configurer et gérer des parties de fléchettes avec plusieurs joueurs, plusieurs équipes, et plusieurs joueurs par équipe.

Modes de jeu à implémenter :

1. Mode X01
Le mode doit permettre de jouer à :
- 301
- 501
- 701
- 901
- ou n’importe quelle autre centaine choisie par l’utilisateur

Fonctionnalités X01 :
- Choisir le score de départ
- Choisir le nombre d’équipes
- Choisir le nombre de joueurs par équipe
- Nommer les équipes
- Nommer les joueurs
- Gérer le tour de jeu équipe par équipe
- À chaque tour, saisir le score total des 3 fléchettes
- Soustraire ce score au total de l’équipe
- Empêcher de descendre sous 0
- Détecter automatiquement la victoire quand une équipe atteint exactement 0
- Afficher :
  - score restant de chaque équipe
  - équipe active
  - joueur actif
  - historique des tours
- Ajouter un bouton “Annuler le dernier tour”

Options X01 à prévoir dans le code :
- double out
- master out
- nombre de manches / legs
Même si ces options ne sont pas complètement codées au départ, prévoir une structure propre pour les ajouter.

2. Mode Cricket
Le mode Cricket doit permettre de jouer sur les cibles :
- 15
- 16
- 17
- 18
- 19
- 20
- Bull

Fonctionnalités Cricket :
- Choisir le nombre d’équipes
- Choisir le nombre de joueurs par équipe
- Nommer les équipes et les joueurs
- Choisir entre deux variantes :
  a) Cricket avec points
  b) Cricket sans points mais avec objectifs avancés

Variante A : Cricket avec points
Règles :
- Chaque cible doit être touchée 3 fois pour être fermée
- Simple = 1 marque
- Double = 2 marques
- Triple = 3 marques
- Si une équipe a fermé une cible et qu’au moins une autre équipe ne l’a pas fermée, les touches supplémentaires sur cette cible rapportent des points
- Une cible fermée par toutes les équipes ne rapporte plus de points
- Victoire :
  - toutes les cibles fermées
  - et score supérieur ou égal aux autres équipes

Variante B : Cricket avancé sans points
Règles :
- Chaque équipe doit fermer toutes les cibles classiques :
  - 15 à 20 + Bull
- En plus, chaque équipe doit réussir :
  - 3 doubles
  - 3 triples
- Les doubles et triples doivent être comptabilisés séparément comme objectifs globaux
- Pas de score de points
- Victoire :
  - toutes les cibles fermées
  - 3 doubles validés
  - 3 triples validés

Interface Cricket :
- Afficher un tableau clair avec les cibles en lignes et les équipes en colonnes
- Pour chaque cible, afficher le nombre de marques : 0, 1, 2, fermé
- Afficher les points si variante avec points
- Pour la variante avancée, afficher aussi :
  - doubles réussis / 3
  - triples réussis / 3
- Prévoir des boutons simples pour enregistrer une action :
  - cible touchée
  - multiplicateur : simple, double, triple
- Gérer le tour actif :
  - équipe active
  - joueur actif
  - passage automatique au joueur suivant après validation du tour
- Ajouter un historique des actions
- Ajouter un bouton “Annuler le dernier tour”

Gestion des équipes et joueurs :
- L’utilisateur doit pouvoir choisir :
  - nombre d’équipes
  - nombre de joueurs par équipe
- Chaque équipe peut avoir plusieurs joueurs
- Le tour doit alterner correctement :
  - équipe 1 joueur 1
  - équipe 2 joueur 1
  - équipe 1 joueur 2
  - équipe 2 joueur 2
  - etc.
- Si les équipes n’ont pas le même nombre de joueurs, le code doit rester robuste.
